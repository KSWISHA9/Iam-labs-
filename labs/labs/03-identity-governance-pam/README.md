# Identity Governance & Privileged Access Management (PAM) Lab

## Objective
Implement identity governance and privileged access controls to reduce risk, enforce least privilege, and support audit readiness in an enterprise IAM environment.

## Environment
- Active Directory
- Microsoft Entra ID
- PowerShell automation

## What This Lab Covers
- Identity Governance concepts (IGA)
- Access certifications and reviews
- Role and entitlement modeling
- Privileged account separation
- Just-In-Time (JIT) access concepts
- Privileged access logging and review

## Key IAM Activities
- Reviewed user and group memberships to identify excessive privileges
- Conducted access certification reviews and documented approvals
- Implemented separation of standard and privileged accounts
- Simulated temporary elevation of privileges using time-bound access
- Reviewed privileged activity for audit and security purposes

## Validation
- Verified removal of unnecessary privileged access
- Confirmed access approvals and revocations
- Reviewed logs and group membership changes
- Validated least-privilege enforcement
