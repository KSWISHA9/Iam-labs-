# Identity Incident Response Lab

## Objective
Investigate and resolve identity-related security and access incidents using structured IAM incident response workflows.

## Environment
- Active Directory
- Microsoft Entra ID
- PowerShell
- Simulated ticketing workflow

## Incident Scenarios Covered
- Account lockouts due to repeated authentication failures
- Access denied issues caused by incorrect group membership
- Excessive privileges identified during access review
- MFA and authentication failures

## Key IAM Activities
- Analyzed authentication and account lockout events
- Identified root causes of identity-related incidents
- Remediated issues by adjusting group membership, permissions, or authentication settings
- Restored user access following least-privilege principles
- Documented incident details, remediation steps, and validation results

## Validation
- Verified successful authentication after remediation
- Confirmed correct access levels for affected users
- Reviewed logs and sign-in activity to ensure resolution
- Ensured no excessive privileges remained
